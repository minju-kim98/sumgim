# 숨김 (Sumgim) — 사내 워크샵 기획안 발표자료

> **워크샵 주제**: 진행 중인 업무 또는 관심 있는 영역에 대해 가지고 있는 문제의식, 불편, 아이디어를 꺼내어 실제 적용 가능한 서비스 기획안을 작성
> **발표 형식**: 5분 발표 + QA
> **컨셉**: 우리 팀이 만들고 있는 '보임(Boim)'에서 영감을 받아 떠올린 짝 도구 '숨김(Sumgim)'의 서비스 기획안

---

## 시간 배분 (총 5분)

| # | 슬라이드 | 시간 | 핵심 메시지 |
|---|---------|------|-----------|
| 1 | Hook | 0:20 | "발표 도중에 Mattermost DM 떠본 적, 있으세요?" |
| 2 | 출발점 — 보임 | 0:40 | 보임을 만들다가 발견한 빈자리 |
| 3 | 문제의식 | 0:50 | 우리가 실제로 겪는 4가지 |
| 4 | 아이디어 — 숨김 | 0:50 | '보임'의 짝으로 떠올린 '숨김' |
| 5 | 핵심 기능 | 1:00 | 회의 자동 감지 + 알림 차단 + 상태 동기화 |
| 6 | 사용자 시나리오 | 1:00 | 한 사람의 하루 — 시작 / 도중 / 끝 |
| — | 마무리 | 0:20 | 한 줄 마감 |

각 슬라이드는 큰 글자 한 줄 + 보조 시각 한 개. 텍스트는 슬라이드에 적지 말고 **말로**.

## 📑 슬라이드 본문

---

### 슬라이드 1 — Hook (0:20)

```
[중앙 큰 글씨]
발표 도중에
Mattermost DM이
떠본 적,
있으세요?
```

> [발화]
> "한 번이라도 있으신 분?" — 손 들어주세요 (5초 대기)
> "저는 매주 한 번씩 있어요. 화면 공유 중인데 갑자기 사적인 DM이 올라오는, 그 순간이요. 오늘은 그 불편을 서비스 기획으로 풀어본 이야기를 드리려고 합니다."

**시각**: 어두운 배경 + 흰 텍스트. 이미지 없이 임팩트.

---

### 슬라이드 2 — 출발점: 보임을 만들다가 (0:40)

```
[제목]
'보임'을 만들다가 발견한 빈자리

✅ 보임(Boim) = 문서·발표 자료를 자동으로 만들고 띄워주는 도구
   → '보여주는' 쪽은 잘 풀려 있다

❌ 그런데 막상 띄우는 순간 —
   Mattermost DM, 캘린더 팝업, 카카오톡이 같이 뜬다
   → '가리는' 쪽은 아무도 안 다루고 있다
```

> [발화]
> "출발점이 분명했어요. 우리 팀이 보임이라는, 자료를 자동으로 만들어서 띄워주는 도구를 만들고 있어요. '보여주는' 쪽은 잘 풀고 있는 거죠.
> 그런데 옆에서 보면서 깨달았어요. 정작 띄우는 순간, 사적인 알림이 같이 뜨면 자료를 아무리 잘 만들어도 의미가 없어요. '가리는' 쪽은 비어 있더라고요. 그래서 그 빈자리를 채우는 짝을 기획해봤습니다."

**시각**: 좌측 '보임' 로고 + 화살표 + 우측 빈 박스(물음표).

---

### 슬라이드 3 — 문제의식 (0:50)

```
[제목]
우리가 실제로 겪는 4가지

🔴 회의·발표 시작할 때 → 방해금지 켜는 걸 까먹는다
🟡 끝나고 나서 → 다시 끄는 것도 까먹는다 (DM 놓침)
🟠 알림 패널 열면 → 내용이 그대로 보인다 (화면 공유 중엔 못 엶)
🔵 Windows 방해금지는 → 전부 ON / 전부 OFF뿐 (Mattermost만 조용히가 안 됨)
```

> [발화]
> "팀원 분들께 가볍게 여쭤봤더니, 이 4개를 다 겪고 계셨어요. Windows에 방해금지 기능이 분명히 있는데도요.
> 특히 두 번째 — 회의 끝나고 다시 켜는 걸 까먹어서 DM을 한참 후에 보는 일, 저만 그런 줄 알았는데 아니더라고요. '나만 그런 게 아니구나'에서 기획이 시작됐습니다."

**시각**: 4개 아이콘 + 한 줄씩.

---

### 슬라이드 4 — 아이디어: 숨김 (0:50)

```
[중앙]
보임(Boim) ↔ 숨김(Sumgim)

보임 = 보여줄 자료를 자동으로 띄운다
숨김 = 띄울 때 가릴 알림을 자동으로 가린다

[하단 작게]
한 사람이 발표할 때, 두 도구가 같이 동작한다
```

> [발화]
> "그래서 짝을 만들었습니다. 이름도 대놓고 짝이에요. 보임은 '보여줌', 숨김은 '숨김'. 보임이 자료를 띄우는 순간, 숨김이 옆에서 알림을 가립니다.
> 같은 사용자, 같은 순간, 정반대 방향의 문제. 한 짝으로 묶었을 때 비로소 완성되는 그림이라고 봤습니다."

**시각**: 보임 로고 ↔ 숨김 로고. 가운데 화면 공유 아이콘.

---

### 슬라이드 5 — 핵심 기능 (1:00)

```
[제목]
숨김이 자동으로 하는 일

1. 회의·발표 감지
   - 디스플레이 복제 (노트북 화면 = 외부 모니터)
   - PowerPoint 슬라이드쇼 진입
   - 단축키 (Ctrl+Alt+M) — 수동 토글

2. 알림 차단
   - OS 방해금지 ON
   - 카카오톡 등 외부 메신저 일시 정지

3. 메신저 상태 동기화
   - Mattermost / Slack 상태 → "회의 중"

4. 끝나면 원상복구 + 놓친 메시지 알림
   - "Mattermost DM 3건, 멘션 1건" 토스트 → 클릭 시 점프
```

> [발화]
> "기획의 핵심은 '자동'이에요. 사용자가 손 댈 일이 없어야 합니다.
> 회의나 발표가 시작된 걸 감지하면 알림을 끄고, Mattermost 상태를 '회의 중'으로 바꾸고, 끝나면 다 원래대로 돌립니다. 끝났을 때 놓친 DM 개수도 알려주고, 클릭하면 바로 그 대화로 점프합니다.
> 자동 감지의 핵심은 '디스플레이 복제'예요. 회의실 모니터에 HDMI를 꽂으면 그 순간을 거의 100% 정확하게 잡을 수 있습니다."

**시각**: 4단계 플로우 다이어그램.

---

### 슬라이드 6 — 사용자 시나리오 (1:00)

```
[제목]
한 사람의 하루로 그려보면

📍 14:00 — 회의실 입장
   → HDMI 연결 → 숨김이 자동으로 '회의 모드' ON
   → Mattermost 상태: "회의 중"
   → OS 알림 OFF / 카톡 일시 정지

📍 14:00–14:50 — 발표 진행
   → 화면 공유 중 어떤 알림도 뜨지 않음
   → 본인은 발표에만 집중

📍 14:50 — 회의 종료
   → HDMI 분리 → 자동으로 원상복구
   → 토스트: "Mattermost DM 3건 · 멘션 1건"
   → 클릭 → 해당 대화로 점프
```

> [발화]
> "기획안이니까 사용자 입장에서 한 번 그려볼게요.
> 14시에 회의실에 들어가요. HDMI를 꽂는 순간 알림이 다 꺼지고, 동료들이 보는 Mattermost 상태도 '회의 중'으로 바뀝니다. 50분 동안 발표에 집중하고, 끝나서 HDMI를 빼는 순간 — 다 원래대로 돌아오고, 그 사이 놓친 메시지를 한 번에 보여줍니다.
> 사용자가 한 일은? HDMI를 꽂고 빼는 것, 그게 다입니다."

**시각**: 타임라인 형태로 3개 시점(14:00 / 진행 중 / 14:50).

## 🗂️ 부록 슬라이드 (QA 대비)

본문엔 안 보여주고, 질문이 들어왔을 때 점프해서 보여줄 수 있는 슬라이드.

---

### A1 — 기술 스택

```
Frontend: React + TypeScript
Backend: Rust (Tauri 2.x)
Win32 API: 디스플레이 감지, 알림 차단
메신저: Mattermost REST API + Slack Web API
배포: GitHub Releases + 서명 자동 업데이트

→ Electron 대신 Tauri를 고른 이유:
   상주 앱이라 메모리(약 30MB vs 약 150MB)가 핵심
```

---

### A2 — '회의'를 어떻게 감지하나

```
3가지 트리거 (사용자가 끄고 켤 수 있음)

1. 디스플레이 복제 모드 — 노트북 화면 = 외부 모니터 (가장 정확)
2. PowerPoint 슬라이드쇼 진입
3. 전체화면 앱 (오탐 위험 → 기본 OFF)

+ 단순 외부 모니터 연결은 자동 ON 하지 않고 "켤까요?" 토스트만
+ 글로벌 단축키 Ctrl+Alt+M — 언제든 수동
```

> [발화 보조]
> "'회의'의 정의가 사람마다 달라서, 사용자가 트리거를 고르게 했어요. 디스플레이 복제는 거의 100%, PPT는 발표 시작 시점, 전체화면은 오탐이 있어서 옵션입니다."

---

### A3 — 프라이버시 / 보안

```
✅ Mattermost / Slack 토큰은 로컬 저장 (사용자 AppData)
✅ 외부 서버 없음 — 사용자 PC ↔ Mattermost / Slack 직통신
✅ 텔레메트리 없음
✅ 자동 업데이트 = 서명 검증
```

---

### A4 — 비슷한 도구는 없나

```
Windows Focus  → 알림 차단은 되지만 메신저 동기화 없음
Mac Focus      → Mac 전용
Viva Insights  → 분석 중심, 실시간 차단 아님

숨김 = '회의 자동 감지 + 메신저 상태 + 자동 복구' 셋을 묶음
```

---

### A5 — 왜 '숨김'이라는 이름인가

```
저희 팀이 만들고 있는 서비스: 보임 (Boim)
→ 보고서·프레젠테이션을 자동으로 만들고 띄우는 도구
→ '보여주는' 도구

숨김 (Sumgim)
→ 띄울 때 가려야 할 알림을 자동으로 가리는 도구
→ '가리는' 도구

화면 공유 = 보여줄 것과 가릴 것이 동시에 발생하는 순간
→ 짝이 되는 도구가 같이 동작하는 게 자연스럽다
```

> [발화 보조]
> "이름의 유래를 물어보시는 분들이 있어서 — 저희 팀이 만드는 보임이 발표 자료를 자동으로 띄워주는 도구잖아요. 그런데 띄우는 순간 사적인 알림이 같이 떠요. 그래서 짝을 만들었습니다. 보임으로 보이고, 숨김으로 가립니다."

## 🎤 발표 톤 가이드

- **문제의식 → 아이디어 → 기획안** 흐름을 명확하게 — 워크샵 주제 그대로
- **남이 아니라 우리** — "팀원 분들도 겪으셨어요", "저만 그런 줄 알았는데"
- **기획안 톤** — 결정된 제품을 자랑하는 게 아니라, 어떤 불편에서 어떤 서비스로 구체화했는지를 보여주는 톤
- **마무리는 한 문장** — "보임으로 보이고, 숨김으로 가립니다"

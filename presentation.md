라이트닝 토크 본문엔 안 보여주고, 질문 받을 때 점프해서 보여줄 수 있는 슬라이드.

### A1 — 기술 스택 (개발자 질문 대비)

```
[제목]
어떻게 만들었나

Frontend: React + TypeScript
Backend: Rust (Tauri 2.x)
Win32 API: 디스플레이 감지, 알림 차단
메신저: Mattermost REST + Slack Web API
배포: GitHub Releases + minisign 서명 자동 업데이트

→ Electron이 아니라 Tauri인 이유: 메모리 ~30MB (Electron은 ~150MB)
→ 상주 앱이라 가벼움이 핵심
```

---

### A2 — 어떻게 회의를 감지하나 (가장 자주 나올 질문)

```
[제목]
'회의'를 어떻게 정의?

3가지 자동 감지 (사용자가 켜고 끔):

1. 디스플레이 복제 모드 (가장 정확) — 노트북 화면 = 외부 모니터
2. PowerPoint 슬라이드쇼 진입
3. 전체화면 앱 (오탐 위험 → 기본 OFF)

+ 단순 외부 모니터 연결 → 자동 ON 안 함, "켤까요?" 토스트만
+ 글로벌 단축키 Ctrl+Alt+M → 언제든 수동
```

> [발화 보조]
> "회의 정의가 사람마다 달라서 사용자가 켜는 트리거를 고르게 했습니다. 디스플레이 복제는 거의 100% 정확하고, PPT는 발표 시작할 때, 전체화면은 오탐 위험이 있어서 옵션이에요."

---

### A3 — 시장 / 수익화 (투자자 질문 대비)

```
[제목]
시장 신호 + 수익화 가설

타겟: 노트북 + 외부 회의 잦은 사무직 (한국 기준 수백만 명)
경쟁: 빈 시장 — 오프라인 회의 자동화는 PoC조차 드묾

수익화 (가설, 검증 전):
- B2C: 오픈소스 (현재) → 무료
- B2B: 팀 단위 — 캘린더 동기화, 관리자 대시보드, 정책 푸시
  → 보안에 민감한 회사 (금융 / 컨설팅) 우선

판단 신호 (베타에서 확인할 것):
- WAU 100명 / 재방문율 50%+ / "OS Focus보다 낫다" NPS 70%+
```

> [발화 보조]
> "지금은 오픈소스로 가설 검증부터 합니다. B2B 모델은 베타 후에 결정할 거고, 보안 민감한 회사들이 가장 빠르게 반응할 거라고 보고 있어요."

## 🎤 발화 톤

- **문제는 모두의 이야기**처럼 — "여러분도 한 번씩…" 아닌 "저도, 여러분도"
